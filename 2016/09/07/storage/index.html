HTML5 Web Storage

Veröffentlicht am 2016-09-07

概述

HTML5 新增了一个非常重要的功能，可以在客户端本地保存数据的Web Storage功能，之前都是使用Cookies在客户端保存简单的用户信息，但是Cookies存在几个问题：

（1）大小限制：Cookies的大小不能超过4KB
（2）带宽：Cookies是随着HTTP请求一起发送到服务器，会占用一部分带宽
（3）复杂性：Cookies在查询，修改时操作复杂。

针对以上问题，HTML5中，重新提供了一种在客户端本地保存数据的功能——Web Storage。具体分为两种： sessionStorage 和 localStorage。几乎所有最新版的浏览器都支持 sessionStorage 和 localStorage存储，包括 IE 浏览器。

sessionStorage 和 localStorage 属于特定的源（协议+域名+端口），Web Storage每一个给定的源（given origin）维持一个独立的存储区域，例如http://www.abc.com 和 http://www.def.com拥有属于自己的sessionStorage 和 localStorage。在某个源的脚本中无法操作其他源的sessionStorage 和 localStorage。
但是cookies只区分协议+域名， 不区分端口。同一协议+域名，不同端口会共享同一份cookies.例如http://www.abc.com:1000 和 http://www.abc.com:2000 会共享cookies。因此如果想要在同一域名不同端口下使用不同享cookies就要加标识来存储，才能保证查询修改时不会影响另外一个端口的数据。

查看

若查看某域的这三者的信息，可以在浏览器控制台中查看（F12 > 选择console控制台）

  • cookies：document.cookie
  • localStorage: localStorage
  • sessionStorage: sessionStorage

语法

localStorage 和 localStorage

sessionStorage 和 localStorage是Storage类的两个实例，自然也具有Storage类的属性和方法。以下操作将Storage换成localStorage或者sessionStorage即可。

//相当于setItem()
localStorage.userId = ‘123456’;
//相当于getItem()
var userId = localStorage.userId；

//返回一个字符串，其中包含了以分号分隔的cookie列表字符串 (即key=value键值对)
document.cookie;

//写一个新cookie,updatedCookie是一个键值对形式的字符串。注意，你只能用这个方法一次设置或更新一个cookie。
document.cookie = updatedCookie;
//举个栗子
document.cookie = “name=oeschger”;
document.cookie = “favorite_food=tripe”;
alert(document.cookie);// 显示: name=oeschger;favorite_food=tripe

//得到名为test2的cookie
document.cookie = “test1=Hello”;
document.cookie = “test2=World”;
var myCookie = document.cookie.replace(/(?:(?:^|.;\s)test2\s\=\s([^;]).$)|^.*$/, “$1”);
alert(myCookie);// 显示: World
```

从以上操作过程可以看出，cookies操作真的很麻烦。没有sessionStorage 和 localStorage简介直观。

区别

  1. cookies数据始终在同源的http请求中携带（即使不需要），即cookies在浏览器和服务器间来回传递。而sessionStorage和localStorage不会自动把数据发给服务器，仅在本地保存。cookies数据还有路径（path）的概念，可以限制cookies只属于某个路径下。
  2. 存储大小限制也不同，cookies数据不能超过4k，同时因为每次http请求都会携带cookies，所以cookies只适合保存很小的数据，如会话标识。sessionStorage和localStorage 虽然也有存储大小的限制，但比cookies大得多，可以达到5M或更大。

  3. 数据有效期不同：

    • sessionStorage表示一个页面会话，sessionStorage 里面的数据在页面会话结束时会被清除。页面会话在浏览器打开期间一直保持，并且重新加载或恢复页面仍会保持原来的页面会话，在当前页面打开同源的新页面（链接打开，或者window.open打开）也会保持原来的页面会话。但是在新标签或窗口打开一个页面会初始化一个新的会话；
    • localStorage：始终有效，窗口或浏览器关闭也一直保存，即使关机也存在，除非用户手动删除；
    • cookies如果设置的有效时间，在设置的cookie过期时间之前一直有效，即使窗口或浏览器关闭，如果没有设置有效时间，那cookies的生命周期就在应用结束（一般为浏览器关闭）的时候结束。注意：浏览器的cookies和服务的session计时方式不一样，session是从无操作开始计时，cookies是从cookies文件生成开始计时。
      (1) 在当前页面刷新
      (2) 从当前页面打开一个同源新窗口（或者window.open()）
      (3) 打开一个新窗口，输入同源的URL
      (4) 关闭该源下所有的页面，再重新打开
      sessionStorage有效范围(1)(2)
      localStorage有效范围(1)(2)(3)(4)
      cookies如果不设置有效时间，有效范围是(1)(2)(3)

  4. 作用域不同，sessionStorage不在不同的浏览器窗口中共享，即使是同一个页面；localStorage 在所有同源窗口中都是共享的；cookie也是在所有同源窗口中都是共享的。

  5. Web Storage 支持事件通知机制，可以将数据更新的通知发送给监听者。
  6. Web Storage 的 api 接口使用更方便。
